## Frontend architecture notes (no schema change)

- **`coaching_activities` is not directly SELECT-able by end users** (table-level GRANT denies it before RLS). The public projection **`coaching_activities_public`** (`title, tier, definition`) is the correct read path for surfacing activity metadata in user-facing views. `CoachingSessionView` was repointed to it (fetch session, then read the activity from the public view, attach as `coaching_activities`). - **Report-commitment DP surface resilience:** `ReportCommitmentsTab` wraps each RPC (`report_list_commitments`, `dp_list_my_report_commitments`) in a `safe` helper (try/catch â†’ `[]`) and renders a My-Development-style empty state instead of an error, so a caller with no commitments (incl. super-admin viewing someone else's report) sees the empty card, not a "couldn't load" error. The shared team-scope sentinel in the add flow stays the literal `"team"` for both team and paired reports; the paired report swaps `Person A`/`Person B` to real first names at the presentation layer only.
